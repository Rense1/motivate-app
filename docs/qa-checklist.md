# REVIVE — 実機テストチェックリスト

## Android テスト（リリース前）

### 起動・認証
- [ ] アプリが正常に起動する
- [ ] スプラッシュ画面が表示される（2秒）
- [ ] ログイン画面が表示される
- [ ] メール/パスワードでログインできる
- [ ] Google ログインボタンが表示される
- [ ] Google ログインをタップすると Chrome Custom Tab が開く
- [ ] Google 認証後にアプリに戻り /home に遷移する
- [ ] ログアウト後に再ログインできる

### ナビゲーション
- [ ] ボトムナビゲーションが機能する（ホーム/目標/設定）
- [ ] 各ページが正常に表示される
- [ ] ページ遷移が滑らか

### 通知
- [ ] アプリ初回起動時に通知許可ダイアログが表示される
- [ ] タスクに通知を設定できる
- [ ] 設定した時刻に通知が届く
- [ ] 通知をタップするとアプリが起動する

### データ
- [ ] 目標を作成・編集・削除できる
- [ ] マイルストーンが正常に動作する
- [ ] タスクの完了状態が保存される
- [ ] アプリを再起動してもデータが保持される
- [ ] オフライン時に適切なエラーが表示される

### UI/UX
- [ ] フォントが日本語で表示される
- [ ] バイブレーション（ハプティック）が動作する
- [ ] モーダルが正常に開閉する
- [ ] スクロールが滑らか
- [ ] キーボードが入力フィールドを隠さない

### パフォーマンス
- [ ] ホーム画面の表示が 3 秒以内
- [ ] メモリリーク（長時間使用後に重くならない）

## ストア審査チェックリスト（共通）

### コンテンツ・機能
- [ ] プライバシーポリシーの URL が機能する
- [ ] アプリ説明が実際の機能と一致する
- [ ] テストアカウントが審査担当者向けに用意されている（Google OAuth使用時は必須）
- [ ] ログイン不要でも基本機能が使える（審査対策）

### Google Play 固有
- [ ] DATA_SAFETY セクションを入力済み（収集データ、共有先）
- [ ] ターゲット年齢層を設定済み
- [ ] アプリアイコンが Google のポリシーに準拠（性的・暴力的要素なし）

### App Store 固有
- [ ] Privacy Manifest（`PrivacyInfo.xcprivacy`）を追加済み（iOS 17.4 以降必須）
- [ ] App Tracking Transparency が不要な場合は使用しないことを明示
- [ ] IPv6 ネットワーク環境でも動作する

---

## 審査落ちしやすいポイント（REVIVE 固有）

| リスク | 対策 |
|--------|------|
| Google OAuth のみでログイン必須 | メール/パスワードも提供済み ✅ |
| Supabase URL がハードコードされていない | `.env.local` で管理済み ✅ |
| 課金機能（プレミアム）の説明 | In-App Purchase を使う場合は Apple の指定 API が必要 |
| 通知のスパム | ユーザーが通知を OFF にできることを確認 |
